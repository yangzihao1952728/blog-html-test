## 一、先看修复了什么（表情 Bug）

**原来的问题**（`app.js` 旧代码）：

```js
const pick = pool[Math.floor(Math.random() * pool.length)]; // ← 随机抽一个!
```

点一下表情按钮，它偷偷从一堆表情里**随机**塞一个进输入框，你根本没法选。这不是“表情功能”，是“抽奖功能”。

**修好后的做法**：点表情按钮 → 弹出一个**面板**，里面有分类标签 + 表情网格，你点哪个就插哪个。下面讲到 `app.js` 时会逐行讲这块新代码。

## 二、整体架构（新手最重要的部分）

整个项目**没有框架、没有编译**，就是「一个 HTML + 一个 CSS + 四个 JS 文件」。关键是搞懂这四个 JS 文件怎么配合：

```
index.html  ← 页面骨架（长什么样）
   │
   ├── css/styles.css      ← 动画、玻璃质感、背景光斑
   │
   └── 按顺序加载 4 个 JS（顺序不能乱!）：
         js/utils.js  →  定义工具函数
         js/store.js  →  数据 + 存储到浏览器
         js/ui.js     →  把数据画成页面 + 处理点击
         js/app.js    →  启动 + 发布框 + 主题 + 表情
```

### 为什么顺序不能乱？

四个文件都用了一个“公共名字空间” `window.Pulse`。可以把它想象成**一个大家共用的工具箱**：

- `utils.js` 往工具箱里放工具（叫 `Pulse.utils`）
- `store.js` 从工具箱拿工具用（`Pulse.utils.uid()`），再往里放数据模块（`Pulse.store`）
- `ui.js` 拿 `Pulse.store` 和 `Pulse.utils` 来用
- `app.js` 三个都用

所以 `utils` 必须最先加载，否则后面的人去工具箱里拿，会发现是空的，程序就崩了。

### 每个文件都包在一个“盒子”里（IIFE）

你会看到每个 JS 文件都是这种结构：

```js
(function () {
  'use strict';
  // ... 所有代码 ...
  window.Pulse = window.Pulse || {};
})();
```

这叫 **IIFE（立即调用函数表达式）**——可以理解为一个**自封的盒子**：盒子里的变量（比如 `pendingImages`）只在盒子里有效，不会污染全局、不会跟别的文件撞名。最后一行把想公开的东西挂到 `window.Pulse` 上，别的盒子就能拿到了。`'use strict'` 是开启“严格模式”，让你写错代码时浏览器更早报警。

## 三、`index.html`（页面骨架 + 样式配置）

这个文件负责“页面长什么样”。只讲几个**对理解逻辑最关键**的点：

### 1. 开头那段“防闪烁”小脚本

```html
<script>
  (function () {
    try {
      var t = localStorage.getItem('pulse_theme');        // ① 读用户上次选的主题
      if (!t) t = matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'; // ② 没有的话跟系统走
      if (t === 'dark') document.documentElement.classList.add('dark');  // ③ 给 <html> 加 dark 类
    } catch (e) { /* localStorage 可能用不了，静默忽略 */ }
  })();
</script>
```

**为什么放在最前面？** 如果等 JS 都加载完再切深色模式，页面会先白一下再变黑（闪烁）。这段代码在页面**渲染前**就跑，直接定好主题，用户感觉不到切换。

- `localStorage` = 浏览器自带的“小存储”，关页面也不会丢，后面存动态数据也靠它。
- `document.documentElement` = 就是 `<html>` 标签。给它加 `dark` 类，整个页面就进深色模式（Tailwind 的规则：`dark:` 开头的样式只在有 `dark` 类时生效）。

### 2. Tailwind 的引入和配置

```html
<script src="https://cdn.tailwindcss.com"></script>
```

这是 **Tailwind 的 Play CDN**——一个在线工具，你写的 `class="rounded-xl bg-white"` 它会现场翻译成真正的 CSS。方便，但只适合演示（控制台会有一条警告，正常）。

下面紧跟一段配置：

```js
tailwind.config = { darkMode: 'class', theme: { extend: { colors: {brand:{...}}, boxShadow: {soft, glow} } } }
```

告诉 Tailwind：“深色模式用 class 方式判断”、“加几个自定义颜色和阴影”。

### 3. 复用样式（组件类）

```html
<style type="text/tailwindcss">
  @layer components {
    .card { @apply rounded-3xl border ... bg-white/80 backdrop-blur-xl ...; }
    .btn-primary { @apply ... bg-gradient-to-r from-indigo-500 ... ; }
  }
</style>
```

`@apply` 的意思是“把后面这些零散的 Tailwind 类打包，起个名字叫 `.card`”。这样 HTML 里写 `class="card"` 就等于写了那一长串，代码更干净。

### 4. 页面结构（三栏布局）

```
<header>          顶部导航栏（Logo、主题切换、发布按钮）
<main> 三栏网格：
  ├─ 左侧栏：当前用户卡片（头像、昵称、获赞数）
  ├─ 中间：  发布框 + 筛选条 + 动态列表（#feed）
  └─ 右侧栏：热门话题、推荐用户
```

关键：**动态列表的容器叫 `#feed`**。后面所有“画动态、处理点赞评论”都围绕这个容器，记住它。

### 5. 脚本按顺序引入

```html
<script src="js/utils.js"></script>
<script src="js/store.js"></script>
<script src="js/ui.js"></script>
<script src="js/app.js"></script>
```

放在 `<body>` 最末尾，因为这时候页面元素都画好了，JS 可以直接操作它们。

## 四、`css/styles.css`（动效与质感）

这是纯 CSS（不能用 `@apply`，因为没经过 Tailwind 处理）。讲三个代表：

### 背景光斑（blob）

```css
.blob { position: absolute; border-radius: 9999px; filter: blur(70px); }
.blob-a { background: radial-gradient(...); animation: blobFloat 20s ... infinite; }
@keyframes blobFloat { 0%,100%{transform:translate(0,0)} 33%{transform:translate(40px,-30px)} ... }
```

几个模糊的彩色圆球在背景里慢慢飘。`@keyframes` 定义“动画的关键帧”，`animation` 让它无限循环。纯装饰，但让页面“有设计感”。

### 点赞心跳动画

```css
.like-btn.is-liked .heart { animation: heartPop .42s ...; }
@keyframes heartPop { 0%{scale:1} 45%{scale:1.45} 70%{scale:.9} 100%{scale:1} }
```

被点赞时，心形会“弹一下”。JS 只负责给按钮加 `is-liked` 类，弹的效果由这段 CSS 自动完成——**JS 管逻辑，CSS 管表现**，各司其职。

### 骨架屏闪烁（shimmer）

```css
.shimmer { background: linear-gradient(...); background-size:200% 100%; animation: shimmer 1.3s infinite; }
```

页面刚加载、数据还没画出来时，显示几个灰条左右扫光，告诉用户“内容马上来”，比干等舒服。

## 五、`js/utils.js`（工具箱）

放各种“纯函数”——给个输入就有输出、不碰页面状态的工具。挑最重要的几个讲：

### 生成唯一 ID

```js
uid() { return 'p' + Date.now().toString(36) + Math.random().toString(36).slice(2, 7); }
```

每条动态、每条评论都需要一个唯一编号。用“当前时间戳 + 一小段随机数”拼出来，基本不会重复。

### 防 XSS 转义（安全相关）

```js
escapeHtml(str) {
  return String(str ?? '').replace(/[&<>"']/g, c => ({'&':'&amp;','<':'&lt;','>':'&gt;','"':'&quot;',"'":'&#39;'}[c]));
}
```

用户发的文字里可能藏着 `<script>` 这种东西。如果直接塞进页面，就会被当成代码执行（危险！）。这个函数把 `<` 变成 `&lt;` 等无害文字，**所有用户输入在画到页面前都必须过这道关**。

`textToHtml` 在它基础上多了两件事：把网址变成可点链接、把回车换成换行 `<br>`。

### 相对时间

```js
formatRelativeTime(ts) {
  const diff = Date.now() - new Date(ts).getTime();   // 算"距今过了多久"（毫秒）
  const s = diff / 1000;
  if (s < 45) return '刚刚';
  if (s < 3600) return Math.floor(s/60) + ' 分钟前';
  ...
}
```

把时间戳变成“刚刚”“3 分钟前”“2 小时前”。这就是页面上时间显示的来源。

### 头像渐变色（关键设计点）

```js
avatarGradient(seed) {
  let h = 0; const s = String(seed);
  for (let i=0;i<s.length;i++) h = (h*31 + s.charCodeAt(i)) >>> 0;  // 把名字算成一个数字
  const palettes = [['#6366f1','#a855f7'], ...];                   // 几套预设配色
  const [a,b] = palettes[h % palettes.length];                      // 用数字取一套
  return `linear-gradient(135deg, ${a}, ${b})`;
}
```

我们没有真实头像图片，就用**名字算出一个固定的渐变色**当头像底色。同样的名字永远是同一种颜色（因为算出来是固定的），看起来还挺统一。

### 图片压缩（避免存储爆掉）

```js
fileToResizedDataURL(file, maxDim=1280, quality=0.82) {
  return new Promise((resolve, reject) => {
    // 用 FileReader 读图 → 画到 <canvas> 上缩小 → 导出成 JPEG
  });
}
```

手机拍的照片好几 MB，直接存进 `localStorage` 会瞬间塞满。这个函数把图缩小到最长边 1280px 再转成 JPEG，体积小很多。`Promise` 是“我现在先答应你，回头做好了叫你”的异步机制——读图、解码都需要时间，不能一步到位。

### 轻提示 toast

```js
toast(message, type) {
  // 在右下角生成一个小提示条，2.2 秒后自动消失
}
```

“发布成功🎉”“评论已发送”那种一闪而过的提示就是它。

## 六、`js/store.js`（数据 + 存储）

这是整个应用的**大脑/数据库**——所有动态数据都归它管，别的地方要用数据都来找它要。

### 数据长什么样

```js
const USERS = { linxia: {id, name:'林夏', handle:'linxia', bio:'...'}, ... };  // 用户名单
const CURRENT_USER_ID = 'linxia';   // "我"是林夏
const SEED = [ {authorId:'chenmo', minsAgo:8, text:'...', images:[...], likes:248, comments:[...]}, ... ]; // 初始演示数据
```

`minsAgo` 是“几分钟前发的”。真正加载时再换算成具体时间戳。

### 种子数据 → 真实状态

```js
function buildSeedState() {
  const now = Date.now();
  const posts = SEED.map(s => ({
    id: uid(),
    authorId: s.authorId,
    time: now - s.minsAgo * 60000,   // 把"几分钟前"换算成真实时间戳
    ...
    comments: s.comments.map(c => ({..., time: now - c.minsAgo*60000}))
  }));
  posts.sort((a,b) => b.time - a.time);  // 按时间倒序（新的在前）
  return { currentUserId: CURRENT_USER_ID, posts };
}
```

### 读取与保存（核心机制）

```js
function load() {
  if (_state) return _state;            // 已经读过了就直接用（缓存）
  let loaded = null;
  try {
    const raw = localStorage.getItem(STORAGE_KEY);   // ① 先看本地有没有存过
    if (raw) { const parsed = JSON.parse(raw); if (parsed.posts) loaded = parsed; }
  } catch (e) { /* 存储坏了就忽略 */ }
  _state = loaded || buildSeedState();  // ② 有就用存的，没有就用种子数据
  return _state;
}

function persist() {
  try { localStorage.setItem(STORAGE_KEY, JSON.stringify(_state)); }  // 把状态转成文本存起来
  catch (e) { utils.toast('本地存储已满', 'error'); }
}
```

**这就是“关掉浏览器再打开，你发的动态还在”的秘密**：每次改动都 `persist()` 存一次，下次加载先 `load()` 读出来。`try/catch` 是“万一出错别让整个程序崩”的保护。

### 对外提供的操作（API）

```js
addPost({text, images})    // 发新动态 → 往数组最前面塞（unshift）→ 存 → 通知
deletePost(id)             // 删动态
toggleLike(id)             // 点赞/取消（liked 取反，数量 ±1）→ 存 → 通知
addComment(postId, text)   // 加评论
```

每个操作都是：**改数据 → `persist()` 存盘 → `emit()` 喊一声“数据变了”**。`emit` 会通知订阅了的人（目前主要是计数刷新）。

## 七、`js/ui.js`（渲染 + 交互）——最重要的文件

负责“把数据画成页面”和“处理用户的点击”。

### 怎么画一条动态（拼 HTML 字符串）

```js
function postCardHTML(post, animate) {
  const u = store.getUser(post.authorId);   // 查作者信息
  const cover = utils.avatarGradient(u.name + u.handle);  // 算头像颜色
  return `
    <article class="card ${animate?'post-enter':''} p-5" data-id="${post.id}">
      <header>...头像、昵称、@handle、时间、更多菜单...</header>
      ${post.text ? `<div ...>${utils.textToHtml(post.text)}</div>` : ''}  ← 正文（已转义）
      ${imageGridHTML(post.images)}                                         ← 配图
      <footer>...点赞、评论、分享按钮...</footer>
      ${commentsBlockHTML(post)}                                            ← 评论区（默认隐藏）
    </article>`;
}
```

注意几个点：

- `data-id="${post.id}"`：把动态 ID 藏在元素上，**后面点击时能反查是哪条动态**。这是很常用的技巧。
- 正文用 `textToHtml`（经过转义），不能直接插原文。
- 配图数量不同，布局不同（`imageGridHTML` 里：`grid-cols-1/2/3/2`，1 张大图、4 张 2×2 等）。

### 渲染整个列表

```js
function renderFeed(animate) {
  const posts = visiblePosts();              // 根据筛选条件拿到要显示的动态
  feed.innerHTML = posts.map(p => postCardHTML(p, animate)).join('');  // 拼成大字符串一次性塞进去
}
```

`innerHTML` 一次替换所有内容，简单粗暴。

### 事件委托（新手重点概念！）

我们**没有**给每个点赞按钮单独绑事件，而是只给 `#feed` 容器绑了一个监听：

```js
feed.addEventListener('click', onFeedClick);
```

为什么？因为动态是动态生成的（可能删、可能加），提前绑按钮绑不到。**事件委托**利用了“事件冒泡”——点哪个按钮，点击事件都会一层层往上冒，冒到 `#feed` 时被抓住。然后在处理函数里用 `e.target.closest('[data-action]')` 判断“点的是不是某个功能按钮、是哪个”：

```js
function onFeedClick(e) {
  const card = e.target.closest('[data-id]');      // 找到被点的动态卡片
  const act = e.target.closest('[data-action]');   // 找到被点的功能按钮
  if (action === 'like') { ... }                    // 点的是点赞 → 处理点赞
  if (action === 'toggle-comments') { ... }         // 点的是评论 → 展开/收起
  if (action === 'share') { ... }
  ...
}
```

**一个监听搞定所有按钮，新增/删除动态都不用重新绑事件。**

### 局部更新（为什么不整体重画？）

点赞时如果调用 `renderFeed()` 把整个列表重画一遍，会有两个问题：① 评论框要是正开着会被收起；② 用户正在评论框打字，一重画输入框没了，字也丢了。所以点赞用**只改这一个按钮**的方式：

```js
if (action === 'like') {
  const liked = store.toggleLike(card.dataset.id);    // ① 改数据
  act.classList.toggle('is-liked', liked);            // ② 改按钮外观（心跳动画靠这个类）
  heart.setAttribute('fill', liked ? 'currentColor' : 'none');  // ③ 心形填色/空心
  cnt.textContent = utils.formatCount(p.likes);       // ④ 更新数字
}
```

只动该动的地方，体验流畅。评论追加（`appendComment`）同理，只往评论列表里加一个 `<li>`，不动其它。

### 图片大图预览（lightbox）

点小图 → 用同一张图的大尺寸铺满屏幕，点空白或按 Esc 关闭。

## 八、`js/app.js`（启动 + 发布框 + 表情）——重点逐行讲

这个文件做三件事：启动、管发布框（文字+图片）、管主题切换。**表情面板是新加的，逐行讲。**

### 启动流程

```js
function init() {
  ui.init();            // 让 ui 模块初始化（画页面、绑事件）
  paintComposeAvatar(); // 给发布框画上"我"的头像
  // ... 绑各种事件 ...
  wireTheme();          // 绑主题切换
  syncComposer();       // 初始化发布按钮状态
}
if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
else init();
```

意思：如果页面还没加载完，就等加载完（`DOMContentLoaded`）再启动；已经加载完就直接启动。

### 发布框基本逻辑

- `syncComposer()`：监听字数，超过 500 变红、禁用发布按钮；空内容也禁用。
- `autoresize()`：输入文字时让文本框自动变高。
- `renderPreviews()` / `onImagesPicked()`：选了图片就压缩、显示缩略图预览，每张右上角有个 × 能删。
- `publish()`：把文字和图片交给 `store.addPost()` 存起来 → 清空发布框 → `ui.prependPost()` 把新动态插到列表最顶上 → 弹“发布成功”提示。

### ★ 表情面板（修复的核心，逐行讲解）

**第 1 步：准备表情数据（按分类）**

```js
const EMOJI = {
  '表情': ['😀','😃', ...],
  '手势': ['👍','👎', ...],
  '自然': ['🌸','🌺', ...],
  // ... 共 8 类
};
let activeEmojiCat = '表情';   // 当前显示哪个分类，默认"表情"
```

这是一个**对象**，键是分类名，值是表情数组。`activeEmojiCat` 记住“现在在看哪一类”。

**第 2 步：创建面板元素，塞进发布框**

```js
const emojiPanel = document.createElement('div');                    // 用 JS 造一个 <div>
emojiPanel.className = 'mt-2 hidden rounded-2xl border ... p-2';     // 给它样式；hidden = 一开始藏起来
composePreview.after(emojiPanel);                                    // 插到图片预览区的后面（发布框内部）
```

`document.createElement` = 用代码造元素；`.after()` = 插到某个元素后面。这个面板一开始带 `hidden` 类（不显示）。

**第 3 步：画面板内容（分类标签 + 表情网格）**

```js
function renderEmojiPanel() {
  // ① 生成分类标签：遍历每个分类名，做成按钮；当前分类高亮（紫色底白字）
  const tabs = Object.keys(EMOJI).map((c) =>
    `<button class="emoji-cat ... ${c === activeEmojiCat ? 'bg-violet-600 text-white' : '...hover...' }" data-cat="${c}">${c}</button>`
  ).join('');
  // ② 生成当前分类的表情网格：每个表情一个按钮，带上 data-emo 记住它是什么表情
  const grid = EMOJI[activeEmojiCat].map((e) =>
    `<button class="emoji-item ..." data-emo="${e}">${e}</button>`
  ).join('');
  // ③ 拼到一起，塞进面板
  emojiPanel.innerHTML = `<div ...>${tabs}</div><div class="grid grid-cols-8 ...">${grid}</div>`;
}
```

**关键技巧**：每个按钮都带了 `data-cat` / `data-emo`，把“它代表什么”写在元素上。后面点击时就能读出来——和动态卡片的 `data-id` 是同一套思路。

**第 4 步：打开/关闭面板**

```js
function toggleEmojiPanel(force) {
  const open = force != null ? force : emojiPanel.classList.contains('hidden'); // 没传参就取反当前状态
  if (open) renderEmojiPanel();            // 打开前先刷新内容
  emojiPanel.classList.toggle('hidden', !open);  // 加/去 hidden 类来显示/隐藏
}
```

`classList.toggle('hidden', !open)`：第二个参数是“强制加还是去”。`!open` 为 true（即要关闭）就加 `hidden`，`false` 就去掉。

**第 5 步：把选中的表情插到光标处**

```js
function insertEmojiAtCursor(emo) {
  const ta = composeText;
  const s = ta.selectionStart, e = ta.selectionEnd;            // 记住光标位置
  ta.value = ta.value.slice(0, s) + emo + ta.value.slice(e);   // 把表情"切"进光标处
  ta.focus();                                                   // 焦点回到输入框
  ta.selectionStart = ta.selectionEnd = s + emo.length;         // 把光标移到表情后面
  autoresize(); syncComposer();                                  // 文本框变高 + 更新字数
}
```

**这就是“插到光标处”的精髓**：`slice(0,s)` 是光标前的文字，`+ emo` 插入表情，`+ slice(e)` 是光标后的文字，三段拼起来。插完把光标放到表情后面，方便继续选下一个。**注意没关闭面板**，所以可以连点好几个。

**第 6 步：绑定事件（在 `init` 里）**

```js
// ① 点表情按钮 → 切换面板开/关；stopPropagation 阻止事件冒泡（否则会被下面的"点外面关闭"立刻关掉）
$('emojiBtn').addEventListener('click', (ev) => { ev.stopPropagation(); toggleEmojiPanel(); });

// ② 面板内部的点击：要么切分类，要么插表情
emojiPanel.addEventListener('click', (ev) => {
  ev.stopPropagation();                                  // 同样阻止冒泡
  const catBtn = ev.target.closest('.emoji-cat');        // 点的是分类标签？
  if (catBtn) { activeEmojiCat = catBtn.dataset.cat; renderEmojiPanel(); return; }  // 切分类并重画
  const item = ev.target.closest('.emoji-item');         // 点的是表情？
  if (item) insertEmojiAtCursor(item.dataset.emo);       // 插入这个表情
});

// ③ 点到面板外面（且不是表情按钮）→ 关闭
document.addEventListener('click', (ev) => {
  if (emojiPanel.classList.contains('hidden')) return;                       // 已经关了就不管
  if (!emojiPanel.contains(ev.target) && !ev.target.closest('#emojiBtn')) {  // 点的不是面板、不是按钮
    emojiPanel.classList.add('hidden');                                      // 关掉
  }
});

// ④ 按 Esc → 关闭
document.addEventListener('keydown', (ev) => {
  if (ev.key === 'Escape' && !emojiPanel.classList.contains('hidden')) emojiPanel.classList.add('hidden');
});
```

> **新手概念——事件冒泡 & stopPropagation**：你点一个按钮，点击事件会从按钮一层层往父级传（冒泡），直到 `document`。所以我们把“点外面关闭”绑在 `document` 上也能收到面板内部的点击。但面板内部的点击我们想自己处理、不想触发“关闭”，就用 `ev.stopPropagation()` 阻止它继续往上冒。这就是为什么面板内点击不会被误关。

## 九、把一切串起来：一次“点赞”从头到尾发生了什么

理解了这个，你就懂了整个项目：

1. 你点某条动态的爱心 → 点击事件**冒泡**到 `#feed`
2. `onFeedClick` 抓住，用 `closest('[data-action]')` 判断出是“点赞”，用 `closest('[data-id]')` 查出是哪条动态
3. 调 `store.toggleLike(动态ID)` → 改 `liked` 和数量 → `persist()` 存进 `localStorage`
4. 回到 UI，**只更新这一个按钮**：加 `is-liked` 类 → CSS 触发心跳动画；数字 +1
5. 下次刷新页面，`store.load()` 从 `localStorage` 读出“已点赞”状态，爱心还是亮的

**这就是整个项目的运转方式：界面触发 → 改数据（store）→ 存盘（localStorage）→ 局部刷新界面。**
